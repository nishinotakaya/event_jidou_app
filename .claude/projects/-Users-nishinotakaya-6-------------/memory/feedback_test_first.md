name: テストファースト
description: 実装後は必ず本番でテストしてから報告する。テストせず「成功」と報告しない
type: feedback
---

実装後は必ず本番環境でテストしてから完了報告すること。ローカルで動いても本番で動かないケースが多い（headless Chromium、セッション、WebSocket等）。

**Why:** 2026-04-12のセッションで、テスト不足のまま「✅成功」と報告 → 実際は空投稿/フリーズ/50人バグ等が多発。ユーザーの信頼を大きく損なった。

**How to apply:**
- デプロイ後、必ず `heroku run` で本番テスト
- UIに反映される値は本番DBで確認
- 「成功」と言う前にユーザーが確認できるURLを提示
- ローカルテストだけで完了報告しない
